# 2-5 보조기억장치와 입출력장치

## ✅ 보조기억장치
- **보조기억장치**는 메모리의 휘발성을 보완하고, 메모리보다 큰 저장 공간을 제공

### 종류
- **하드 디스크 드라이브 (HDD, Hard Disk Drive)**: 자기적인 방식
- **플래시 메모리 기반 저장장치**: 전기적인 방식
    - USB 메모리, SD 카드, SSD(Solid-State Drive)

## ✅ RAID (Redundant Array of Independent Disks)
- **RAID**는 보조기억장치에 저장된 정보를 안정적이고 안전하게 관리하는 방법으로, 여러 개의 독립적인 보조기억장치를 하나의 보조기억장치처럼 사용
- 목적: `데이터의 안정성` or `성능` 확보
- 하드 디스크나 SSD로 RAID 구성 가능

### RAID 레벨
- **RAID0, RAID1, RAID2, RAID3, RAID4, RAID5, RAID6**이 대표적인 RAID 레벨입니다.

### RAID0
- 데이터를 여러 보조기억장치에 단순하게 나누어 저장하는 구성 방식
    - **스트라입(stripe)**: 마치 줄무늬처럼 분산되어 저장된 데이터
    - **스트라이핑(striping)**: 분산하여 저장하는 동작
    - 장점: 빠른 입출력 속도
    - 단점: 저장된 정보가 안전하지 않음

### RAID1
- 완전한 복사본을 만들어 저장하는 구성 방식 (미러링이라고도 함)
    - 장점: 복구가 간단하고 안정성이 높음
    - 단점: 복사본이 저장된 크기만큼 사용 가능한 용량 적어짐

### RAID4
- 패리티 정보를 저장하는 디스크를 따로 두는 구성 방식
    - **패리티(parity)**: 오류를 검출할 수 있는 정보
    - 장점: 오류 검출용 장치를 따로 두면 RAID1에 비해 적은 하드 디스크로도 안전하게 데이터 보관 가능
    - 단점: 패리티를 저장하는 장치에 병목 현상 발생

### RAID5
- 패리티를 분산하여 저장하는 구성 방식
    - **패리티 분산 저장**으로 RAID4의 병목 현상을 보완 가능

### RAID6
- 서로 다른 2개의 패리티를 두는 구성 방식
    - 장점: 안전성 높음 (2개의 패리티로 오류를 검출하고 복구 가능)
    - 단점: 새로운 정보를 저장할 때마다 2개의 패리티가 저장되어 쓰기 속도가 느림

## ✅ 입출력 기법

### 장치 컨트롤러와 장치 드라이버
- **장치 컨트롤러**: CPU와 입출력장치 사이의 통신을 중개하는 하드웨어
    - 모든 입출력장치는 각자의 장치 컨트롤러를 통해 연결되어 정보를 교환
- **장치 드라이버**: 장치 컨트롤러의 동작을 알고, 장치 컨트롤러가 컴퓨터 내부와 정보를 주고받을 수 있도록 하는 프로그램

### 입출력 작업을 수행하는 방법
- **프로그램 입출력**: 프로그램 속 명령어로 입출력 작업을 수행하는 방법
- **인터럽트 기반 입출력**: 다중 인터럽트
    - 여러 입출력장치로부터 인터럽트가 동시다발적으로 발생하는 경우 처리
    - **프로그래머블 인터럽트 컨트롤러 (PIC)**: 여러 장치 컨트롤러에서 보낸 하드웨어 인터럽트 요청들의 우선순위를 판별하여 CPU에게 알려주는 장치
- **DMA(Direct Memory Access) 입출력**
    - CPU를 거치지 않고 입출력장치와 메모리가 직접 상호작용하는 방식
    - **DMA 컨트롤러**가 시스템 버스에 연결되어 있음

### PCIe (Peripheral Component Interconnect Express)
- **PCIe**: PCI의 발전된 형태로, SSD, GPU, 네트워크 인터페이스 카드 등 다양한 입출력장치를 연결
